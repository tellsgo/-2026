 第 1 次课｜从零开始：WSL2、Ubuntu 与第一个贪吃蛇（静默自学版）

  课堂形式
  本课程采用静默自学模式：无口述讲解、无实时演示。教师仅提供计时与文字提示；学生按步骤独立完成。

  ---

  学习目标
   - 在 Windows 上完成 WSL2 + Ubuntu 的可用环境配置
   - 在 Ubuntu 中成功运行一个最小可用的贪吃蛇程序
   - 形成第一份完整的实验记录（含截图与关键命令）

  ---

  课前准备（学生自查）
   - Windows 10/11 系统，确认虚拟化已开启（在 BIOS 或任务管理器中显示"已启用虚拟化"）
   - 稳定的网络连接；若校园网络受限，建议准备手机热点作为备用
   - C 盘预留至少 20GB 可用空间

  ---

  国内网络使用提示
   - Microsoft Store 访问不稳定时，可使用系统命令安装 WSL2（详见指导书）
   - 包下载缓慢时，优先使用校园网；必要时可切换至国内镜像源

  ---

  课堂流程（静默）
   1. 阅读指导书相关章节 —— 10 分钟
   2. 安装 WSL2 与 Ubuntu —— 50 分钟
   3. 完成贪吃蛇任务并运行 —— 50 分钟
   4. 整理截图与记录 —— 20 分钟
   5. 提交与自检 —— 10 分钟

  ---

  操作步骤（按顺序执行）
   1. 按指导书步骤启用 WSL2 功能并安装 Ubuntu
   2. 首次进入 Ubuntu 后完成基础更新，安装常用工具（git、python、pip），并完成 iFlow 配置
   3. 创建课程工作目录（例如 ~/research-lab/lesson1）
   4. 打开 Cursor（或其他编辑器）连接 WSL2，创建新项目
   5. 使用提示词生成贪吃蛇最小可运行代码（控制台或 pygame 皆可）
   6. 运行程序并截图：
      - 若 GUI 显示异常，先确认 WSLg 是否启用，再检查相关依赖
   7. 将关键命令、截图与问题记录整理到实验记录模板中

  ---

  产出与提交
   - 贪吃蛇程序运行截图或短录屏
   - 实验记录文档（包含环境版本、关键命令、问题与解决方案）

  ---

  自检清单
   - [ ] 能够正常进入 Ubuntu 终端
   - [ ] python --version 命令可用，pip 命令可用
   - [ ] 贪吃蛇程序能够启动并响应键盘操作

  ---

  常见问题
  安装失败：优先检查系统版本与虚拟化设置

  下载缓慢：切换网络或使用国内镜像源

  图形界面不可用：确认 WSLg 支持状态及 Windows 更新状态

  磁盘附加错误（"无法将磁盘 C:\Program Files\WSL\system.vhd 附加到 WSL2，系统找不到指定的文件"）：VHDX
  文件可能在重启后被系统自动删除。可先检查并清理残留文件，再重新运行安装程序
