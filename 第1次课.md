# 第 1 次实验课优化｜零基础入门 WSL2、Ubuntu 与贪吃蛇开发

## 一、实验目标

1. 在 Windows 系统下，独立完成 WSL2 与 Ubuntu 的环境搭建。
2. 掌握基本命令行操作、软件包管理与开发工具配置。
3. 能在 Ubuntu 环境下开发并运行一个“最小可用版贪吃蛇”程序（命令行版/GUI均可）。
4. 撰写完整、规范的实验报告，包括关键命令、环境版本、实际运行截图/录屏、遇到的问题与解决方法、技术反思与展望。

---

## 二、课前准备自查

请严格自查以下条件，提前发现与排除障碍，确保课堂高效：

- **硬件与OS：**
    - Windows 10/11 家庭版/专业版（建议保持系统为最新版，已激活）。
    - BIOS/UEFI 已开启虚拟化（Task Manager > 性能 > 虚拟化 应为“已启用”）。
    - C 盘可用空间≥20GB。

- **网络要求：**
    - 稳定外网（如校园网受限，须备热备份：如手机热点）。
    - Microsoft Store 可正常访问（或准备命令行离线安装方案）。
    - 能科学切换/配置国内源。

- **工具：**
    - 推荐安装 Windows Terminal 和 VSCode（或 Cursor、WSL兼容编辑器）。
    - 带有截图/录屏能力的软件，如 Snipaste、QQ/微信截图、Windows自带录屏等。

## 三、实验进程与分阶段任务建议

### **A. 指导材料预读（10min）**

- 提前阅读指定指导书章节。 对关键术语（WSL、WSLg、pip源、目录结构）标注重点。
- 可选自查任务：脑海中思考，一旦出错该如何报告、如何截图与溯源。

---

### **B. 环境搭建（50min）**

**1. WSL2、Ubuntu 安装与检查**
- 按官方(或教师补充版)步骤，依次执行：
    ```powershell
    # 启用 WSL2
    dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
    dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
    wsl --install
    # 若失败，采用离线包或教师专用镜像
    ```
- **强烈建议**每一步截图留存，出错时动态反馈并保留完整报错内容。
- **首启 Ubuntu 后**立即设置用户名（建议学号/英文名）、密码（记录且安全保存）。

**2. Ubuntu 基础配置与工具链安装**
- 系统更新与开发环境快速部署（推荐教学用标准流程，用一条命令块保存到实验记录）：
    ```bash
    sudo apt update && sudo apt upgrade -y
    sudo apt install -y git python3 python3-pip
    ```
- 配置国内 pip 源，防止后续下载安装超时：
    ```bash
    mkdir -p ~/.pip
    echo -e "[global]\nindex-url = https://pypi.tuna.tsinghua.edu.cn/simple" > ~/.pip/pip.conf
    ```

**3. 项目目录规范化**
- 推荐文件结构：
    ```
    ~/research-lab/lesson1/
        snake.py
        README.md
        result.png  (截图文件)
    ```
- 统一命名利于实验考核与代码归档。

**4. 编辑器与开发同步**
- 必须实现本地与WSL2下代码同步（推荐 VSCode/Remote WSL or Cursor）。
- 鼓励用 Git 初始化仓库并做首次提交，后续便于问题回溯。

---

### **C. 项目编码与运行（50min）**

**1. 贪吃蛇最小实现（控制台/GUI皆可）**
- 提供简版样例代码（教师可根据班级实际水平即可）。
    - 控制台版本无需额外依赖，简明易调试。
    - GUI 版本（pygame等）需要`pip install pygame`，重申 WSLg 环境判别（`echo $WAYLAND_DISPLAY`等）。
- 鼓励改进和功能扩展，如添加“高分记录”、“自动结束检测”或“更友好的键位支持”等。

**2. 程序运行、验证与录屏/截图**
- 明确演示要求：“需有人机交互演示（键盘控制），建议录屏、大图分步截图”。
- 指定截图要求，包括：
    - 成功运行的界面（含完整窗口，不遮挡）。
    - 错误与异常现象的界面。
    - 关键命令行或版本信息（如`python3 --version`、`lsb_release -a`界面）。

### **E. 自查清单与高频问题库**

#### **自查清单**
- [ ] WSL2 & Ubuntu 可正常进入（有提示符）。
- [ ] `python3 --version` / `pip3 --version` 正常输出，无报错。
- [ ] pip更换国内源（如遇拉取速度慢，有无配置并复查）。
- [ ] 贪吃蛇程序可无误启动，响应键盘（如WASD/方向键）控制。
- [ ] 上传的实验文档齐全，截图内容完整可读，文件命名规范。

#### **高频问题与处理建议（建议先查FAQ再求助老师）：**

| 问题类别 | 常见症状 | 推荐解决方案 |
|-----|---------|--------------|
| 安装失败 | WSL不可用，提示系统版本不符 | 检查 Windows 版本，BIOS虚拟化，必要时手动升级 |
| pip/apt 下载慢 | 卡在 XX% 或超时 | 切换 pip、apt 国内源 |
| GUI 界面不出 | pygame挂死，窗口闪退 | 检查WSLg支持（Windows 11/WSL2），确认`echo $WAYLAND_DISPLAY`有输出；更新WSL组件 |
| 依赖报错 | ModuleNotFoundError | 仔细检查 pip 安装、python/pip 版本是否一致 |
| 权限问题 | Permission denied | 加 `sudo` 重试，或者检查当前目录权限 |
| 录屏/截图导出困难 | 找不到截图 | 学会使用 Win+Shift+S、截图和录屏工具 |
