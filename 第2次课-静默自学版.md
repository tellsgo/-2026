# 第 2 次课｜工具链升级：Cursor 高阶、Kimi/iflow、MCP 与 Gitee（静默自学版）

> 课堂无口述、无演示。教师仅提供计时与文字提示；学生按步骤独立完成。

## 目标
- 完成 AI 编程工具链配置并稳定可用。
- 建立代码托管与版本同步流程。
- 形成工具链配置记录与问题清单。

## 课前准备
- 已完成第 1 次课环境搭建。
- 准备 Cursor、Kimi 或 iflow、Gitee 账号（或学校统一账号）。
- 准备 API Key（如无则使用网页版作为替代）。

## 国内网络提示
- 访问国外站点不稳定时，优先使用 Gitee 镜像或课程提供的安装包。
- 大文件下载慢时，先用校园网或同学共享离线包。

## 课堂流程（静默）
- 阅读工具链清单 10 min。
- 配置 Cursor 与 WSL2 连接 30 min。
- 配置模型与提示词并完成用模型生成一个功能为输入两个数，输出它们的和的Python程序的小任务 40 min。
- 建立 Gitee 仓库与首次推送 30 min。
- 记录问题与解决 20 min。

## 操作步骤（照做即可）
1. 安装并登录 Cursor，检查能正常打开 WSL2 项目目录。
2. 配置模型（Kimi/iflow/其他）并完成一次“生成-运行-修复”小任务。
3. 建立课程代码目录（如 `~/research-lab/lesson2`），初始化 Git。
4. 在 Gitee 创建远程仓库并完成首次推送。
5. （可选）按指导书配置 MCP；若失败，记录原因并继续后续任务。
6. 整理工具链配置表：账号、版本、关键设置、常见报错。

## 产出与提交
- 工具链配置清单截图或文字记录。
- Gitee 仓库地址与一次成功提交记录。

## 自检清单
- Cursor 能打开并编辑 WSL2 文件。
- 能正常调用模型完成一次改错。
- 本地 Git 与远程 Gitee 可互相同步。

## 常见问题
- 登录失败：检查网络/验证码，必要时使用手机热点。
- 模型无法调用：确认 API Key 与地区限制，必要时改用网页版。
- Git 推送失败：检查远程地址与认证方式（Token/SSH）。
